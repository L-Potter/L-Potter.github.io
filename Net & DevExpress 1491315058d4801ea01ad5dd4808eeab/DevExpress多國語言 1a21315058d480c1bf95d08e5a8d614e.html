There are several steps to implement localization in C# using resource files.

  1. Create a new .NET standard project with the name i18N (or any name you prefer).
  1. Right-click on your project and add a new Resource file (.resx) by doing the following: Select Add > New Item, then pick Resource file and name it resources.resx

3. Double-click on resources.resx file and click Add Resource and add your keys with values. You can also make Access Modifier as Public if you want to make it global. Finally, save your resource.resx file.

4. The resource.resx file is the default resource file. Now you need to add a language-specific resource file with language code. Then add another Resource file with the name resource.de.resx for German language support and add Keys with Values. In this way, you can add your all Resource files with these naming conventions like resource.fr.resx, resource.it.resx, and so on.

當你呼叫 SetCulture("zh-TW")，C# 會自動載入 Resources.zh-TW.resx 中的對應文字，這是 .NET 本地化 (Localization) 的標準行為。

✅ SetCulture("zh-TW") 如何對應 Resources.zh-TW.resx？

當你設定 Thread.CurrentThread.CurrentUICulture = new CultureInfo("zh-TW") 時：

  1. C# 會自動尋找對應的 .resx 檔案
    • 如果設定為 zh-TW，C# 會嘗試載入 Resources.zh-TW.resx。
    • 如果找不到 zh-TW 版本，就會回退到 Resources.resx (預設語言，例如英文)。
  1. 使用 Properties.Resources 來存取資源
    • 程式碼中存取的資源名稱不變，但 顯示的值會根據 CultureInfo 變化。

✅ 實作步驟

📌 1. 建立 Resources.resx（英文預設）

NameValue (英文)
BUDDY_NAMEBuddy Name
SAVE_BUTTONSave

📌 2. 建立 Resources.zh-TW.resx（繁體中文）

NameValue (繁體中文)
BUDDY_NAME夥伴名稱
SAVE_BUTTON儲存

📌 3. 建立 SetCulture 方法

using System.Globalization;
using System.Threading;

public void SetCulture(string cultureCode)
{
    Thread.CurrentThread.CurrentUICulture = new CultureInfo(cultureCode);
    Thread.CurrentThread.CurrentCulture = CultureInfo.CreateSpecificCulture(cultureCode);
}

📌 4. 在 Form_Load 內設定語系

private void Form1_Load(object sender, EventArgs e)
{
    SetCulture("zh-TW"); // 設定為繁體中文

    // 設定 UI 文字（會根據文化區域自動選擇對應 .resx）
    btnSave.Caption = Properties.Resources.SAVE_BUTTON;
    lblBuddyName.Text = Properties.Resources.BUDDY_NAME;
}

🔹 如果 CultureInfo 是 "zh-TW"，則 Properties.Resources.SAVE_BUTTON 會自動載入 "儲存"。

🔹 如果 CultureInfo 是 "en-US"（預設），則 Properties.Resources.SAVE_BUTTON 會顯示 "Save"。

✅ SetCulture("zh-TW") 切換時會發生什麼？

當 SetCulture("zh-TW") 被呼叫時：

  1. C# 會先尋找 Resources.zh-TW.resx。
  1. 如果找不到對應的 Key，則回退到 Resources.resx（預設語言）。
  1. 畫面上的 btnSave.Caption 會變成 "儲存"，而不是 "Save"。

✅ 結論

✔ 透過 SetCulture("zh-TW")，C# 會自動對應 Resources.zh-TW.resx 內的翻譯內容。

✔ 你不需要修改程式碼，只要維護 .resx 檔案，應用程式就能支援多語系。

✔ 如果找不到 Resources.zh-TW.resx，C# 會回退到 Resources.resx（通常是英文）。

🚀 這就是 .NET 的多語系支援，簡單又強大！ 🎌

是的，在 C# 的 資源檔 (.resx) 本地化 (Localization) 機制中，通常會使用 ISO 639-1 語言代碼 和 ISO 3166-1 國家代碼 作為命名規則。這樣可以讓應用程式根據 使用者的語言環境 (CultureInfo.CurrentUICulture) 自動載入對應的語系資源。

✅ .resx 檔案的命名規則

在 Visual Studio 中，你可以建立不同語言的 .resx 檔案，例如：

語言.resx 檔名
預設語言 (英文)Resources.resx
繁體中文Resources.zh-TW.resx
簡體中文Resources.zh-CN.resx
日文Resources.ja-JP.resx
德文Resources.de-DE.resx
📌 命名規則：
  • zh-TW 👉 zh (語言) + TW (國家)
  • ja-JP 👉 ja (語言) + JP (國家)
  • 這是基於 ISO 639-1 (語言代碼) + ISO 3166-1 (國家代碼)。

✅ 如何使用 .resx 進行多語系切換

📌 1. 在 Resources.resx 添加鍵值

打開 Resources.resx，然後加入鍵值：

NameValue (英文)
BUDDY_NAMEBuddy Name
SAVE_BUTTONSave

🔹 建立 Resources.zh-TW.resx (繁體中文) 並加上對應的翻譯

NameValue (繁體中文)
BUDDY_NAME夥伴名稱
SAVE_BUTTON儲存

📌 2. 讓程式自動選擇對應語言

using System.Globalization;
using System.Threading;

public void SetCulture(string cultureCode)
{
    Thread.CurrentThread.CurrentUICulture = new CultureInfo(cultureCode);
    Thread.CurrentThread.CurrentCulture = CultureInfo.CreateSpecificCulture(cultureCode);
}

📌 3. 在程式中使用 Properties.Resources

btnSave.Caption = Properties.Resources.SAVE_BUTTON;
lblBuddyName.Text = Properties.Resources.BUDDY_NAME;

當 CultureInfo.CurrentUICulture 設定為 zh-TW 時，Properties.Resources.SAVE_BUTTON 會自動顯示 "儲存"，而不是 "Save"。

✅ 結論

✔ .resx 檔案應該遵循 ISO 639-1 + ISO 3166-1 命名規則 (Resources.zh-TW.resx)，以便 C# 自動識別語系。

✔ 程式碼只需要存取 Properties.Resources.鍵值，C# 會根據 CultureInfo 自動載入對應語系資源。

✔ 你可以用 SetCulture("zh-TW") 來切換語系，讓 UI 內容變成繁體中文。

🚀 這樣就能輕鬆實現多語系支援！ 🌏

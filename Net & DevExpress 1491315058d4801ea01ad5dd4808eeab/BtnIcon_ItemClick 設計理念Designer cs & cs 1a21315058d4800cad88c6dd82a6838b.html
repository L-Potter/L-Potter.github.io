BtnIcon_ItemClick 設計理念Designer.cs & .cs

將 Form1_Load 放在 Form1.Designer.cs，而 BtnSave_ItemClick 放在 Form1.cs，更符合 DevExpress 和 .NET Windows Forms 的標準寫法。這樣可以讓設計碼（UI 設定）與邏輯碼（事件處理）分開，符合 MVC（Model-View-Controller） 風格，使程式碼更清晰且易於維護。

✅ 標準寫法

📌 Form1.Designer.cs（負責 UI 設定）

partial class Form1
{
    private DevExpress.XtraBars.BarManager barManager1;
    private DevExpress.XtraBars.Bar bar1;
    private DevExpress.XtraBars.BarButtonItem btnSave;

    private void InitializeComponent()
    {
        this.barManager1 = new DevExpress.XtraBars.BarManager();
        this.bar1 = new DevExpress.XtraBars.Bar();
        this.btnSave = new DevExpress.XtraBars.BarButtonItem();

        // 設定 barManager
        this.barManager1.Bars.Add(this.bar1);
        this.barManager1.Items.Add(this.btnSave);

        // 設定 BarButtonItem
        this.btnSave.Caption = "保存";
        this.btnSave.Glyph = Properties.Resources.icon_save;
        this.btnSave.ItemClick += new DevExpress.XtraBars.ItemClickEventHandler(this.BtnSave_ItemClick);

        // 設定 Bar
        this.bar1.ItemLinks.Add(this.btnSave);

        // 載入事件
        this.Load += new System.EventHandler(this.Form1_Load);
    }
}

🔹 說明

  • InitializeComponent() 是 Windows Forms UI 設定的標準方法。
  • btnSave.ItemClick += new ItemClickEventHandler(this.BtnSave_ItemClick); 用於事件綁定，但事件邏輯本身放在 Form1.cs。
  • 圖標 btnSave.Glyph = Properties.Resources.icon_save; 在 UI 初始化時設定。

📌 Form1.cs（負責邏輯與事件處理）

using System;
using System.Windows.Forms;
using DevExpress.XtraBars;

public partial class Form1 : Form
{
    public Form1()
    {
        InitializeComponent();
    }

    private void Form1_Load(object sender, EventArgs e)
    {
        // 可以寫入載入時的其他初始化邏輯
        MessageBox.Show("窗體載入完成！");
    }

    private void BtnSave_ItemClick(object sender, ItemClickEventArgs e)
    {
        MessageBox.Show("你點擊了保存按鈕！");
    }
}

🔹 說明

  • Form1.cs 處理 業務邏輯，避免 UI 設定與邏輯混在一起。
  • Form1_Load 事件可額外執行初始化邏輯。
  • BtnSave_ItemClick 只處理按鈕點擊事件，不涉及 UI 設定。

✅ 這樣寫的優勢

  1. 符合 .NET 和 DevExpress 標準風格
    • UI 設定放在 .Designer.cs，邏輯放在 .cs。
    • DevExpress Windows Forms 設計器 預設就是這樣運作的。
  1. 可讀性更好，易於維護
    • UI 和事件邏輯分開，開發者可以更快找到問題。
    • 如果 UI 變更，不會影響 Form1.cs 內的邏輯程式碼。
  1. 避免 Visual Studio 設計器覆蓋你的程式碼
    • 若 Form1_Load 寫在 .Designer.cs，可能會被設計器重新生成時刪除。
    • 事件處理函數（如 BtnSave_ItemClick）應該放在 .cs，避免 UI 設計變更影響它。
